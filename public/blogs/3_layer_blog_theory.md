一、TOFU（认知层） 目的：教育认知、抓问题流量、建立初信任

Definitions 术语解释 / Beginner’s Guides 新手入门 / FAQs 常见问答 目标：Rank for definition/question-based queries；把“行业黑话”翻译成人话。 写法：一词一页 + 交叉内链；每条加外部权威引用与更新时间戳（EEAT）。

Trending Content 热点 / Infographics 信息图 目标：吸社媒分享、抢短期展现。 写法：图表先行，文本服务图表；标题含时间/地区。

Pillar Pages 支柱页 目标：建立主题权威，串起后续所有集群文。 写法：10000 字不重要，“回答得完不完整”才重要；H2/H3 结构要像目录。

Lists 清单 / Best-Of Posts 榜单 目标：承接“xxx list / best xxx”搜索；给用户“第一口答案”。 写法：清单清晰，优选标准透明；内链到具体产品/方案。

看板指标（GSC）：曝光（Impressions）↑、非品牌词覆盖↑、这些页的 CTR 稳步提升。

二、MOFU（评估层） 目的：把“知道问题的人”引到你的方案上

How-To Guides 操作指南 / Steps 步骤 / Checklists 检查表 目标：Rank “how to/step-by-step”；让用户“照着做就行”。 写法：每步一个动词；配流程图、可下载 checklist。

Alternatives 替代方案 / Reviews 评测 / Best-Of/Comparison 对比 目标：承接“X vs Y / alternatives to X”；协助购买决策。 写法：显式列评价维度表（功能/价格/适用人群/限制），别只夸自己。

Case Studies 案例 / Interviews 访谈 目标：社会证明 + 专家背书；增强 EEAT。 写法：用“问题-方案-结果-证据”四段式，放量化指标和截图。

Templates 模板 / Free Tools 免费小工具 目标：Immediate value→留邮箱→后续 nurture。 写法：先能用，后精致；表单前置要克制，免费门槛越低越好。

看板指标：这些页的“查询词→落地页”匹配度、滚动深度、二跳率；下载/留资量。

三、BOFU（转化层） 目的：把评估中的用户送到成交闭环

Product Pages 产品页 目标：承接交易词（size/price/spec）；驱动询盘或下单。 写法：参数表、应用场景、FAQ、评价与证据；Schema（Product/Review/FAQ）必须上。

Landing Pages 着陆页（活动/广告/地域/行业版） 目标：单一 CTA 转化（试用/报价/预约 demo）。 写法：只服务一个意图；折线/卡片复用 MOFU 的证据。

Press Releases 新闻稿 目标：媒体提及 + 可索引证据；给“品牌词+news”一个官方锚点。 写法：简洁、事件驱动；附带媒体可引用的事实和数据。

看板指标：品牌词点击、转化率、咨询表单完成率、商务回呼率。

四、把 23 种内容塞进一张速配表 认知：Definitions / Beginner’s / FAQs / Trending / Infographics / Pillar / Lists / Best-Of 目的：教育与曝光 → 目标：覆盖信息搜索、拉新展示与分享

评估：How-To / Steps / Checklists / Reviews / Alternatives / Case Studies / Interviews / Templates / Free Tools 目的：建立选择标准 → 目标：下载、留资、邮件课订阅

转化：Product / Landing / Press Releases 目的：成交闭环 → 目标：询盘、试用、下单

五、7 天落地行动（小团队照抄） D1：选一个能吃 3 个月的 Pillar 主题（如“industrial-robots-selection”），写 1 张大纲。 D2-D3：从 GSC/第三方库抓 30 个“how-to / list / vs / alternatives”关键词，分 TOFU/MOFU/BOFU。 D4：上线 1 篇 Pillar + 2 篇 How-To + 1 篇 Alternatives；全部加作者页、引用与 FAQ Schema。 D5：做 1 个 Template 或 Free Tool（哪怕是 Excel 模板），配简易下载页。 D6：把产品页补齐：参数表、案例卡片、CTA；建立从 MOFU → 产品页的内链。 D7：开 GSC 看板：按三层分别看 Impressions/CTR/Queries→Page，两周后做第一次回写。 六、常见坑（别踩） 把 BOFU 写成百科：产品页成了“教材”，没有 CTA，没有证据，转化必低。 只写评测不写模板/工具：没有“即时价值”，很难留住邮箱。 热度驱动全站：Trending 能吸量，但不堆“常青内容”= 没复利。 标题冲突：两篇都抢“best xx”，内耗；主题库里应设唯一核心词。

把这 23 种类型当作“兵器库”：TOFU 把你抬进赛道；MOFU 帮用户做选择；BOFU 让选择变成交。
