# 路线图与激励系统 (Roadmap & Incentives) - 场景测试

## 测试目标

验证用户激励机制（签到、分享奖励）以及产品路线图的展示。

## 前置条件

- [ ] 服务已启动
- [ ] 已登录测试账号
- [ ] 测试账号处于未签到状态

---

### 场景 1: 每日签到 (Daily Check-in)

**目标**: 验证签到功能及其阶梯奖励逻辑。

#### 测试步骤
1. [ ] 在页面顶部 Header 找到签到按钮（日历图标）
2. [ ] 点击签到按钮弹出面板
3. [ ] 点击"Check In"
4. [ ] 验证弹出奖励动效
5. [ ] 验证积分余额增加（第一天应 +10）
6. [ ] 再次点击签到按钮

#### 预期结果
- [ ] 签到成功后按钮状态变为"已签到" (Checked)
- [ ] 连续签到天数增加
- [ ] 再次点击无法重复签到，显示今日已完成
- [ ] 如果是第 5/10/30 天，应获得更高的阶梯奖励

---

### 场景 2: 分享奖励 (Share Reward)

**目标**: 验证通过分享作品获得积分奖励。

#### 测试步骤
1. [ ] 访问任意一个公开作品或 OC 详情页
2. [ ] 点击分享按钮弹出分享菜单
3. [ ] 选择一个社交平台（如 Twitter/X）进行分享
4. [ ] 分享完成后触发回调
5. [ ] 检查积分余额

#### 预期结果
- [ ] 每日第一次有效分享应获得 +10 MC 奖励
- [ ] 弹出奖励成功的提示
- [ ] 同一天内第二次分享不再获得奖励，且界面上有相应提示

---

### 场景 3: 产品路线图 (Roadmap)

**目标**: 验证 Roadmap 页面的内容展示与交互。

#### 测试步骤
1. [ ] 访问 `/roadmap` 页面
2. [ ] 浏览不同状态的任务：
    - 已完成 (Completed)
    - 进行中 (In Progress)
    - 计划中 (Planned)
3. [ ] 验证对路线图项的点赞/投票功能（如有）

#### 预期结果
- [ ] 路线图按阶段清晰展示
- [ ] 用户可以了解产品的最新动态和未来计划
