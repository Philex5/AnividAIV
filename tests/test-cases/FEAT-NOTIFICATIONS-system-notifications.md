# FEAT-NOTIFICATIONS 系统通知 - 测试用例

**关联需求**: FEAT-NOTIFICATIONS（PRD）

## 主路径

1. 登录后 Header 展示铃铛入口
   - 期望：有未读时显示数字徽标；无未读时不显示徽标
2. 打开通知面板
   - 期望：展示最新通知列表与“Mark all read/See all”按钮
3. 点击通知跳转
   - 期望：跳转到 `action_url` 并标记已读

## 错误/边界

1. 未登录访问通知 API
   - 期望：返回 `UNAUTHORIZED`
2. 标记不存在的通知
   - 期望：返回 `NOT_FOUND`
3. 通知列表为空
   - 期望：显示空态文案与重试入口（如有）
4. 全站公告投放策略
   - 期望：所有用户都能看到同一公告，各自维护独立读状态
5. 批量标记已读
   - 期望：传入多个 ids 时全部更新为已读状态
6. 分页加载
   - 期望：使用 cursor 正确加载下一页，无重复数据
7. 过期通知
   - 期望：已过期的通知不再出现在列表中
8. 轮询频率验证
   - 期望：未读数量每30秒刷新一次

## 回归

1. 生成完成后仍能正常显示生成结果页
2. 会员订阅状态变化不影响通知列表加载
3. Header 其他入口（主题/语言/用户菜单）不受影响
